# Plan 2.1 Summary: Signup Form Integration

## Objective
Connect the registration form in `signup.html` to the backend `/api/register` endpoint and handle user feedback.

## Changes
- Updated `frontend/js/signup.js` to use `fetch` for sending registration data to the backend.
- Implemented real-time error handling to show backend validation messages (like "Email already registered").
- Added success redirection to `login.html`.

## Files Touched
- `frontend/js/signup.js`

## Verification
- Signup form now sends POST request to `http://localhost:8000/api/register`.
- Backend success returns 200 and triggers redirect.
- Backend error (400) displays error message via `showError`.

## Risks/Debt
- CORS issues might occur if frontend and backend ports differ (handled by CORSMiddleware in main.py).
